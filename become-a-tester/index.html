Help us validate ProductivitySurge macOS apps before App Store release.

ProductivitySurge ships focused, local-first Mac tools. Testers help prove sandboxed folder access, first-run flows, and day-to-day reliability before general release.

How to join

  1. Install the TestFlight app from the Mac App Store if you do not already have it.
  2. Open the invite link for the app section below (or email support if the beta is full).
  3. Accept the invite, install the listed build, and work through the test focus items.
  4. Send feedback to support@productivitysurge.com with macOS version, steps, and screenshots.

What to test

Clean launch

Confirm the app does not silently open or access ~/Documents/Markdown AI on first launch.

Choose Folder

Authorize a folder with the system picker, then create, edit, and save a Markdown file there.

Quit & relaunch

Quit completely, relaunch, and confirm access to the authorized folder returns without choosing again.

  • Optionally try Add Root, Remove Root, and Reauthorize Root.
  • Confirm File → Open works for a separately selected file.
  • Check Help links for Website and Support if present in your build.

How to report issues

Email support@productivitysurge.com and include:

  • App name and build
  • macOS version and Mac model
  • Expected vs actual behavior
  • Steps to reproduce
  • Screenshots or short screen recordings if useful
